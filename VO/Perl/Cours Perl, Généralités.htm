Quelques généralités sur le langage.

Tout programme perl est stocké dans un fichier texte qui commence par une ligne indiquant au système ou se trouve l'interpréteur. Dans un système linux, cet ligne est de la forme:
#!/usr/bin/perl

Si le fichier est rendu exécutable au moyen de la commande unix :

chmod u+x nom.ext

Alors, il suffira, sur la ligne de commande, de faire référence au fichier pour que le programme soit automatiquement exécuté. La manière la plus simple pour savoir si l'interpréteur Perl est installé sur un système est de taper la commande :

perl -v

Au niveau de la ligne de commande.Si on obtient la reponse : 

c:\progs> perl -v
This is perl, version 5.004

Copyright 1987-1997, Larry Wall
Macintosh port Copyright 1991-1998, Matthias Neeracher


Perl may be copied only under the terms of either the Artistic
License or the GNU General Public License, which may be found
in the Perl 5.0 source kit.
c:\progs>

L'interpréteur est bien installé et pret à fonctionner.

La documentation.

La documentation en ligne de Perl (le format POD, Plain Old Documentation) a été conçue par Larry Wall afin de faciliter la documentation des modules et des programmes perl 

\progs> perl perldoc -f pop

pop ARRAY

pop Pops and returns the last value of the array, shortening the
array by one element. Has an effect similar to

$ARRAY[$#ARRAY--]

If there are no elements in the array, returns the undefined
value (although this may happen at other times as well). If
ARRAY is omitted, pops the "@ARGV" array in the main program,
and the "@_" array in subroutines, just like "shift".
c:\progs>

L'appel de l'interpréteur.

La référence à l'interpréteur se fait, sous une console du système au niveau du prompteur.
Supposons que nous disposons d'un programme 'hello.pl' qui se contente d'imprimer la chaîne de caractères 'Hello.', on le lancera comme suit : 

c:\prog> perl hello.pl
Hello.
c:\prog>

Quel que soit le système d'exploitation.
