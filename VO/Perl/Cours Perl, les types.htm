Les types.

Types de données.

Perl considère trois types de données principaux.
Un type simple : Le Scalaire.
Deux types organisés : La liste indexée et le hash.

Selon le contexte dans lequel il est ulilisé, un scalaire sera considéré comme :
Une chaine de caractères.
Une valeur numérique.
Une variable booléenne.
Dans ce dernier cas, la conventions sera :
Une valeur numérique 0 (zéro) ou une chaîne vide ("") Représentent la valeur logique 'Faux'
Toute autre valeur sera évaluée à 'Vrai'.

C'est le premier caractère d'un nom d'identificateur qui va permettre de déterminer à quel type de données est rattaché l'objet en question.
Ce caractère sera :
Un $ (dollard) si l'identificateur représente un scalaire.
Un @ (arobar) si l'identificateur représente une liste.
Un % (pour cent) si l'identificateur représente un hash.

Par ailleurs, chaque type d'objet possède sa propre zone d'identification. Ceci implique que l'on peut voir cohabiter dans la même application :
Le scalaire $a,
La liste @a,
Le hash %a,
Sans que ne se pose le moindre problème au niveau de leur différenciation.
